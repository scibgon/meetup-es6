Introducción

¿Qué es ES6?

{JS}

ECMAScript es el estándar que define cómo debe de ser el lenguaje Javascript.

Javascript es una marca comercial propiedad de Oracle y antes de Sun Microsystems. Cuando los ingenieros de Netscape incorporaron un lenguaje de Scripting en su navegador decidieron, después de varios nombres, decidieron denominarlo Javascript bajo consentimiento de Sun.

Más tarde Netscape decidió proponer su lenguaje como estándar y el trabajo fue realizado por ECMA, dando lugar a ECMAScript. Por tanto, hoy Javascript sería un dialecto de ECMAScript.

Novedades

Let

{JS}

let permite declarar variables limitando su alcance (scope) al bloque, declaración, o expresión donde se está usando. La diferencia con palabra reservada var es que se define una variable global o local en una función sin importar el ámbito del bloque.

                
                  //ES5
                  function varTest() {

                    var x = 31;

                    if (true) {
                      var x = 71;  // misma variable!
                      console.log(x);  // 71
                    }

                    console.log(x);  // 71
                  }
                

                
                  //ES6
                  (function() {

                    let x = 'Hola Santi';
                    
                    if(true) {
                        let x = 'Hola Jose';
                    }

                    console.log(x);  //Hola Santi
                  })();

Const

{JS}

Las constantes presentan un ámbito de bloque tal y como lo hacen las variables definidas usando la instrucción let, con la particularidad de que el valor de una constante no puede cambiarse a través de la reasignación, y no se puede redeclarar. Es necesario inicializar la constante.

                
                  const ALIAS; //Error

                  const ALIAS = "El bicho";
                  console.log(ALIAS) //El bicho

                  ALIAS = "El niño de los Palacios"; //Error

Operador spread

{JS}

El operador spread permite que una expresión sea expandida en situaciones donde se esperan múltiples argumentos (llamadas a funciones) o múltiples elementos (arrays literales)

                
                  function dice(personaje, ...palabras) {
                    return personaje + 'dice: ' + palabras.join('');
                  }
                  console.log(
                    dice('CR7', 'soy','el','mejor','del','mundo.',' Siiiiiiiu' )
                  );

Template Strings

{JS}

Son literales de texto que habilitan el uso de expresiones incrustadas. Es posible utilizar cadenas de texto de más de una línea, y funcionalidades de interpolación de cadenas de texto con ellas.

                
                  function presentation(name, age){
                    return `

                            
Hola! Me llamo ${name} y tengo ${age + 3} años

                          
`;
                  }

                  presentation("Sergio", 26);

Estructuras de datos: Maps

{JS}

Los Mapas son una estructura de datos que almacenan pares de clave (key) y valor (value), son similares a los objetos.

                
                  let map = new Map();

                  map.set('foo', 123);

                  let user = {userId: 1};

                  map.set(user, 'Alex');

                  map.get('foo'); //123

                  map.get(user); //Alex

                  map.size; //2

                  map.has('foo'); //true

                  map.delete('foo'); //true

                  map.has('foo'); //false

                  map.clear();

                  map.size; //0

Estructuras de datos: Maps vs Object

{JS}
  • Un Objeto tiene un prototipo, por lo tanto hay claves por defecto.
  • Las claves de un Objeto son Strings, mientras que pueden ser valores de cualquier tipo para un Mapa.
  • Puede obtenerse fácilmente el tamaño de un Mapa mientras que debe mantenerse manualmente el trazado del tamaño de un Objeto.

Estructuras de datos: Maps vs Object

{JS}

Estas preguntas pueden ayudarte a tomar una decisión:

  • ¿Las claves son desconocidas hasta el tiempo de ejecución? ¿Necesita localizarlas dinámicamente?
  • ¿Tienen todos los valores el mismo tipo, y pueden usarse de forma intercambiable?
  • ¿Necesita claves que no sean strings?
  • ¿Serán añadidos o eliminados a menudo los pares clave-valor?
  • ¿Tiene una cantidad arbitraria (fácilmente modificable) de pares clave-valor?
  • ¿Será iterada la colección?

Estructuras de datos: Set

{JS}

Los sets son conjuntos de elementos no repetidos, que pueden ser tanto objetos, como valores primitivos.

Tiene métodos equivalentes a un Map, con la diferencia que utilizamos add para añadir elementos, y de que en un set las keys y los values son lo mismo, el valor del objeto. Del mismo modo, .entries() devuelve una pareja [value, value]

                
                  let set = new Set();

                  set.add('foo');

                  set.add('bar');

                  set.size //2

                  for(let item of set.entries()){
                    console.log(item);
                  }
                  
                  //["foo", "foo"]
                  //["bar", "bar"]

Desestructuración

{JS}

Las asignaciones de desestructuración te permiten asignar las propiedades de un arreglo o un objeto a variables, usando una sintaxis similar a la utilizada con valores literales

                  
                    //ES5
                    var someArray = [4, 8, 5];

                    var first = someArray[0],
                        second = someArray[1],
                        third = someArray[2];


                  
                  
                
                  
                    //ES6
                    var someArray = [4, 8, 5];

                    var [uno, dos, tres] = someArray;

Clases

{JS}

Son una mejora sintáctica sobre la herencia basada en prototipos de JavaScript. Proveen una sintaxis mucho más clara y simple para crear objetos y lidiar con la herencia.

                  
                    //ES5

                    //Class creation
                    function Document(title, author, isPublished) {
                      this.title = title;
                      5 / 55
                      this.author = author;
                      this.isPublished = isPublished;
                    }

                    Document.prototype.publish = function publish() {
                      this.isPublished = true;
                    };

                    //Class inherittance
                    function Book(title, author, topic) {
                      Document.call(this, title, author, true);
                      this.topic = topic;
                    }

                    Book.prototype = Object.create(Document.prototype);


                  
                  
                
                  
                    //ES6
                    
                    //Class creation
                    class Document {
                      constructor(title, author, isPublished) {
                        this.title = title;
                        this.author = author;
                        this.isPublished = isPublished;
                      }

                      publish(){
                        this.isPublished = true;
                      }
                    }

                    //Class inherittance
                    class Book extends Document{
                      constructor(title, author, topic){
                        super(title, author, true);
                        this.topic = topic;
                      }
                    }

                    let libro = new Book("Gomorra", "Roberto Saviano", "Novela");

Módulos

{JS}

Con export / import podemos tener un sistema de módulos en nuestra aplicación. En la actulidad ningún navegador a implementado la funcionalidad, pero la podemos utilizar incorporando transpiladores como Babel.

                  
                    // module "my-module.js"
                    export function cube(x) {
                      return x * x * x;
                    }

                    const foo = Math.PI + Math.SQRT2;

                    export { foo };

                    //app.js
                    import { cube, foo } from 'my-module';

                    console.log( cube(3) ); // 27

                    console.log( foo );    // 4.555806215962888

Arrow Functions

{JS}

La expresión de función flecha dispone de una sintaxis más corta comparada con la expresión de función convencional y no vincula su propio this, arguments, super, o new.target. Las funciones flecha siempre son anónimas. Estas funciones son funciones no relacionadas con métodos y no pueden ser usadas como constructores.

                  
                    //ES5
                    
                    //Ejemplo 1
                    var sum = function(num1, num2) {
                      return num1 + num2;
                    };

                    //Ejemplo 2
                    var f = function(x){
                      if( x % 2 === 0 ){
                        return x;
                      }else {
                        return 0;
                      }
                      
                    }

                  
                  
                
                  
                    //ES6

                    //Ejemplo 1
                    var sum = (num1, num2) => { return num1 + num2; }
                    sum(2,5); //7
                    
                    //Ejemplo 2
                    const f = x => (x % 2) === 0 ? x : 0;

Proxy

{JS} 
                
                  //sintaxis
                      
                  let proxy = new Proxy(target, handler);

El objeto Proxy se utiliza para personalizar los comportamientos de operaciones básicas como:

  • Observación de propiedades
  • Asignaciones
  • Enumeraciones
  • Funciones invocación
  • etc

Esto lo hace interceptando las llamadas a dichas operaciones y manipulando la respuesta.

Esta funcionalidad no es soportada por los transpiladores. Tiene que estar implementada por el navegador para poder utilizarla.

Proxy: Métodos del handler

{JS}
  • handler.getPrototypeOf(target): Intercepta Object.getPrototypeOf.
  • handler.setPrototypeOf(target, prototype): Intercepta Object.setPrototypeOf.
  • handler.isExtensible(target): Intercepta Object.isExtensible.
  • handler.preventExtensions(target): Intercepta Object.preventExtensions.
  • handler.getOwnPropertyDescriptor(target, property): Intercepta Object.getOwnPropertyDescriptor.
  • handler.defineProperty(target, property, descriptor): Intercepta Object.defineProperty.
  • handler.has(target, property): Intercepta el operador in.
  • handler.get(target, property, receiver): Intercepta el getter de las propiedades.
  • handler.set(target, property, value, receiver): Intercepta el setter de las propiedades.
  • handler.deleteProperty(target, property): Intercepta el operador delete.
  • handler.ownKeys(target): Intercepta Object.getOwnPropertyNames.
  • handler.apply(target, thisArg, argumentsList): Intercepta la función call.
  • handler.construct(target, argumentsList, newTarget): Intercepta el operador new.

Proxy: Ejemplo

{JS} 
                  
                    let validator = {
                      set: function(obj, prop, value) {
                        if (prop === 'age') {

                          if (!Number.isInteger(value)) {
                            throw new TypeError('The age is not an integer');
                          }

                          if (value > 200) {
                            throw new RangeError('The age seems invalid');
                          }
                        }

                        // The default behavior to store the value
                        obj[prop] = value;
                      }
                    };

                    let person = new Proxy({}, validator);

                    person.age = 100;

                    console.log(person.age); // 100

                    person.age = 'young'; // Throws an exception

                    person.age = 300; // Throws an exception

Promesas

{JS}

El objeto Promise (Promesa) es usado para computaciones asíncronas. Una promesa representa un valor que puede estar disponible ahora, en el futuro, o nunca.

                  
                    function hacerAlgoPromesa() {
                      return new Promise( function(resolve, reject){
                        console.log('hacer algo que ocupa un tiempo...');
                        setTimeout(resolve, 3000);
                      })
                    }

                    hacerAlgoPromesa()
                      .then( function(resultado ) {
                        console.log("la promesa terminó");
                    })
